- **Download friction**: Extra step vs web
- **Platform differences**: iOS vs Android audiences
- **Demonstrating value**: Hard to show app experience on web

## App Store Optimization (ASO)

### App Store Listing Elements

**App Name (30 chars iOS, 50 chars Android):**
- Brand + keyword if space
- "AppName - Task Manager"

**Subtitle (30 chars iOS):**
- Key benefit or feature
- "Simple Daily Planning"

**Short Description (80 chars Android):**
- Hook + value prop
- "Plan your day in seconds. Simple tasks, big results."

**Full Description:**
1. Opening hook (first 1-3 lines visible)
2. Key features with emoji bullets
3. Social proof
4. Call to action

### App Store Screenshots

**Order:**
1. Hero/value prop screenshot
2. Key feature 1
3. Key feature 2
4. Key feature 3
5. Social proof/reviews
6. Call to action

**Best Practices:**
- Show the app in use, not just UI
- Add captions explaining benefit
- Consistent visual style
- Real content, not placeholder
- Device frame optional but common

### Keywords Strategy

**iOS (100 char field):**
- No spaces after commas
- No duplicates from title
- Singular/plural both not needed
- No competitor names

**Android (indexed from description):**
- Natural keyword placement
- Repeat important keywords 3-5x
- Long-tail phrases
