# Accessibility Audit Workflow

Audit accessibility of: `$ARGUMENTS`

## Steps

### 1. Define Audit Scope
From `$ARGUMENTS`, identify:
- **Component**: Specific UI component
- **Page**: Full page audit
- **Flow**: Multi-page user journey
- **Full app**: Comprehensive audit

### 2. Run Automated Tests
```bash
# Playwright accessibility
npx playwright test --grep @a11y

# axe-core CLI
npx @axe-core/cli <url>
```

### 3. Manual Review Checklist

**Perceivable:**
- [ ] Text alternatives for images
- [ ] Captions for media
- [ ] Color not sole information carrier
- [ ] Contrast ratios (4.5:1 text, 3:1 large)

**Operable:**
- [ ] Keyboard accessible
- [ ] No keyboard traps
- [ ] Skip links present
- [ ] Focus visible
- [ ] No timing issues

**Understandable:**
- [ ] Language declared
- [ ] Labels for inputs
- [ ] Error identification
- [ ] Consistent navigation

**Robust:**
- [ ] Valid HTML
- [ ] ARIA used correctly
- [ ] Status messages announced

### 4. Classify Findings

| Level | WCAG | Impact | Priority |
|-------|------|--------|----------|
| A | Must fix | Blocker | P0 |
| AA | Should fix | Significant | P1 |
| AAA | Nice to have | Enhancement | P2 |

### 5. Generate Report
Document findings with:
- Issue description
- WCAG criterion violated
- Location (component/selector)
- Remediation steps
- Severity level

### 6. Create Remediation Tasks

**Automatically create ohno tasks for each violation** using MCP tools based on WCAG level:

| WCAG Level | Priority | Task Type | Action |
|------------|----------|-----------|--------|
| A | P0 | bug | Always create (critical barrier) |
| AA | P1 | bug | Always create (significant impact) |
| AAA | P2 | bug | Optional (enhancement) |

For each A/AA violation, use ohno MCP `create_task`:

```
create_task({
  title: "A11y: [issue description]",
  description: "[Description]\n\nWCAG: [criterion number and name]\nElement: [selector or component]\nImpact: [who is affected]\nRemediation: [steps]",
  task_type: "bug",
  estimate_hours: [0.5-2 based on complexity]
})
```

**Example task creation for violations:**
- Missing alt text (WCAG 1.1.1 Level A) → `create_task("A11y: Add alt text to product images", type: bug)` with P0 priority note
- Insufficient contrast (WCAG 1.4.3 Level AA) → `create_task("A11y: Fix contrast ratio on secondary buttons", type: bug)` with P1 priority note
- Missing focus indicator (WCAG 2.4.7 Level AA) → `create_task("A11y: Add visible focus indicator to nav links", type: bug)` with P1 priority note

After creating tasks, report summary:
```
Created [N] accessibility remediation tasks:
- [task-id]: A11y: [name] (Level A/AA)
- ...

WCAG 2.2 AA Compliance: [X]% (estimated after fixes)
```
