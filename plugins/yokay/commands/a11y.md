# Accessibility Audit Workflow

Audit accessibility of: `$ARGUMENTS`

## Steps

### 1. Define Audit Scope
From `$ARGUMENTS`, identify:
- **Component**: Specific UI component
- **Page**: Full page audit
- **Flow**: Multi-page user journey
- **Full app**: Comprehensive audit

### 2. Run Automated Tests
```bash
# Playwright accessibility
npx playwright test --grep @a11y

# axe-core CLI
npx @axe-core/cli <url>
```

### 3. Manual Review Checklist

**Perceivable:**
- [ ] Text alternatives for images
- [ ] Captions for media
- [ ] Color not sole information carrier
- [ ] Contrast ratios (4.5:1 text, 3:1 large)

**Operable:**
- [ ] Keyboard accessible
- [ ] No keyboard traps
- [ ] Skip links present
- [ ] Focus visible
- [ ] No timing issues

**Understandable:**
- [ ] Language declared
- [ ] Labels for inputs
- [ ] Error identification
- [ ] Consistent navigation

**Robust:**
- [ ] Valid HTML
- [ ] ARIA used correctly
- [ ] Status messages announced

### 4. Classify Findings

| Level | WCAG | Impact | Priority |
|-------|------|--------|----------|
| A | Must fix | Blocker | P0 |
| AA | Should fix | Significant | P1 |
| AAA | Nice to have | Enhancement | P2 |

### 5. Generate Report
Document findings with:
- Issue description
- WCAG criterion violated
- Location (component/selector)
- Remediation steps
- Severity level

### 6. Create Remediation Tasks
```bash
npx @stevestomp/ohno-cli create "A11y: Fix [issue]" -t bug
```
